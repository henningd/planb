## Rollen sind das Rückgrat

Eine Rolle ist eine Funktion, kein Mensch. „IT-Leitung" ist eine Rolle — sie kann von Frau Meier besetzt sein und morgen von Herrn Schmidt, ohne dass Verträge oder Pläne neu geschrieben werden. Genau das macht Rollen so mächtig in Krisensituationen.

Erreichbar über die Sidebar **„Notfallhandbuch → Rollen"**.

## Die fünf Pflichtrollen

Diese fünf Rollen werden durch die Onboarding-Prüfung erzwungen:

| Rolle | Wofür im Ernstfall |
|---|---|
| **Geschäftsführung** | Letzte Entscheidung, Außenkommunikation, juristische Vertretung. |
| **Notfallbeauftragte/r** | Leitet den Krisenstab, ruft Pflichtmeldungen ab, dokumentiert. |
| **IT-Leitung** | Technische Sofortmaßnahmen, Wiederanlauf-Reihenfolge, Dienstleister-Kontakt. |
| **Datenschutzbeauftragte/r** | Bewertet, ob DSGVO-Meldung in 72 Std. nötig ist. |
| **Kommunikation** | Texte für Mitarbeiter, Kunden, Presse, Behörden. |

## Hauptperson und Stellvertretung

Pro Rolle gibt es genau **eine Hauptperson** und beliebig viele **Stellvertretungen**. Wenn die Hauptperson nicht erreichbar ist (Urlaub, Krankheit, eigener Schaden), übernehmen die Stellvertretungen in der angegebenen Reihenfolge.

Auf der Detail-Seite einer Rolle sehen Sie alle aktuell zugeordneten Personen, können sie sortieren (für die Stellvertretungs-Reihenfolge), hinzufügen, entfernen.

## Eigene Rollen

Über das System hinaus können Sie **eigene Rollen** anlegen — z. B. „Werkstatt-Pool", „Außendienst-Koordination", „Empfangs-Vertretung". Diese Rollen sind nicht „pflichtbesetzt", können aber genauso mit Personen und Stellvertretungen verknüpft werden.

Eigene Rollen sind nützlich, weil System-Aufgaben (siehe Aufgaben-Inbox) entweder einer Rolle oder einer konkreten Person zugewiesen werden können. Wenn die Rolle wechselt (z. B. wegen Urlaubsvertretung), bleiben die Aufgaben automatisch korrekt zugewiesen.

## RACI

Bei System-Aufgaben gibt es vier Zuordnungsarten:

- **R = Responsible** — verantwortlich für die Ausführung.
- **A = Accountable** — verantwortlich für das Ergebnis (oft Geschäftsführung).
- **C = Consulted** — wird konsultiert, hat Fachwissen.
- **I = Informed** — wird informiert, muss nichts tun.

Im Risiko-Register und bei Aufgaben können Sie pro RACI-Slot eine **Rolle, eine Person oder einen Dienstleister** eintragen. Genau eine Hauptperson pro Slot, beliebig viele Stellvertretungen.

## Wer was sehen darf

Rollen sind für **alle Team-Mitglieder lesbar**. Bearbeiten dürfen sie **Admin und Owner**.

> **Praxis-Hinweis**: Wenn Sie sich unsicher sind, wer die fünf Pflichtrollen besetzen sollte — fragen Sie die Geschäftsführung. Das ist eine Klärung, die im Ernstfall sowieso ansteht; jetzt im ruhigen Moment ist sie billig.
